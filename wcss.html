wCSS rafraîchit vos pages web

Introduction

wCSS est un framework CSS dont l'objectif est d'être le plus simple et le plus rapide possible à prendre en main. wCSS met en forme automatiquement un maximum de balises HTML. Pour les cas où la mise en forme automatique n'est pas possible ou pas préférable, il faut alors simplement ajouter les classes CSS nécessaires pour obtenir la mise en forme souhaitée.

wCSS propose également des outils pour échafauder vos grilles. 1, 2, 3, 4, 5, 6, 8, 10, 12 ou 16 colonnes : vous aurez le choix. Cela n'est pas suffisant ? Vous pouvez ajouter vos propres tailles en toute simplicité !

Cette page de documentation est bâtie sur wCSS, adjointe d'une feuille de styles supplémentaire pour rendre sa mise en page unique. Utilisez les outils de votre navigateur préféré pour l'inspecter !

Merci aux auteurs des outils/sites suivants qui m'ont grandement servi pour bâtir wCSS : Normalize.css, knacss, Code Guide by @mdo, flatuicolors.co, flatuicolorpicker.com.

Grilles .wgrid

Le système de grille CSS est intégralement basé sur PocketGrid, fait par Arnaud Leray. Vous pouvez vous référer à sa documentation pour apprendre à jouer avec.

Quelques ajustements et améliorations ont été apportés dans wCSS pour vous aider à construire vos gabarits aussi vite que possible. Voici ce que vous devez savoir :

  • Vous démarrez une grille avec un .wgrid. C'est comme une ligne (souvent ciblée par .row), mais pas seulement,
  • Les colonnes sont des .wbox,
  • La grille est "mobile-first", donc .wbox démarre avec une largeur de 100%,
  • Vous pouvez définir toutes vos largeurs en pourcentages.

Grilles uniformes

Dans les grilles uniformes, toutes les boîtes ont la même taille. Vous définissez la taille des boîtes depuis l'élément .wgrid.

<div class="wgrid wgridX">
	<div class="wbox">...</div>
	...
</div>

Grilles inégales

Dans une grille inégale, les boîtes d'une même ligne ont des tailles différentes. Les largeurs des boîtes ne sont pas définies depuis .wgrid mais sur chaque .wbox avec une classe .wbox-x-x.

<div class="wgrid">
	<div class="wbox wbox-1-3">...</div>
	<div class="wbox wbox-2-3">...</div>
</div>

Assistants

Cette section rassemble des classes visuelles et utilitaires pratiques pour compléter votre échafaudage et alléger vos feuilles de styles.

Mise en garde : attention aux abus de ce type de classes ! L'overdose de classes appliquées sur des éléments qui sont répétés de nombreuses fois au sein de vos pages va alourdir votre code HTML (en plus de le rendre illisible). Exemple avec une galerie d'images : il est préférable dans ce cas d'utiliser une seule classe dédiée appliquée à chaque image.

Proposition de méthode : choisissez de vous limiter à 4 classes maximum par élément. Si vous devez dépasser cette limite, c'est qu'il est nécessaire de créer une classe pour votre élément.

Espacement

Les assistants d'espacement permettent d'agir sur les marges (internes et externes). Ils sont la concaténation de trois éléments :

  1. p, m = padding, margin
  2. a, t, r, b, l = all, top, right, bottom, left
  3. n, xs, s, m, l, xl = note, extra small, small, medium, large, extra large

Les valeurs des tailles sont les suivantes :

n xs s m l xl
0 5px 10px 20px 40px 80px

Exemples :

  • .man, .pan signifient "margin all none" et "padding all none",
  • .mtl signifie "margin top large",
  • .prxl signigie "padding right extra large".
